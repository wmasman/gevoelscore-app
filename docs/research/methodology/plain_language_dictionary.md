## Purpose

Per [`translation_to_audience.md`](translation_to_audience.md) §2 input #5 + §5.10 + §9.5: the patient-audience track of every Stage T translation MUST render technical terms from the source artefact into plain-language equivalents the audience persona can actually parse, OR provide inline definitions. The dictionary is the single source of truth for those equivalents; per guide #6 §6.3 conflict rule, when the source artefact uses a term differently from a prior dictionary entry, the source wins and the dictionary updates with source-scope-noted alternative meanings.

Per guide #6 §9.6 lock-gate item #3: a patient-audience track-file CANNOT lock if a technical term it uses lacks a dictionary entry or inline definition. The dictionary is the structural precondition for patient-audience-track locks.

## Schema

Each row: **technical term (source vocabulary)** + **Dutch plain-language equivalent** + **English plain-language equivalent** + **source-scope note** (when the equivalent applies only in a specific construct / topic / cluster scope) + **date added**.

When a term is used differently across sources, multiple rows exist (one per scope), per guide #6 §6.3.
