## 2. What fired and why

### Layer 1 fires (SCRIBE 2016 items 3-5, 14, 18, 22-24 + STROBE 2007 items 6, 12, 13)

**L1.1 — Substantive: descriptive-only tier taxonomy attributed to sources that do not lock it.** §9.2 uses a three-tier taxonomy (bootstrap-CI-viable n>=10 / descriptive-only-viable 5<=n<10 / narrative-only-viable 1<=n<=4) and cites "parent Stage -1 audit §5 threshold" as the anchor. The parent Stage -1 audit §5 references "n=5, narrative-only" **once**, in §7's locked-decisions table, applied specifically to dropping the 14-day window; it does not formalise a three-tier viability threshold. Parent MD §7.10 similarly documents "descriptive-only per parent MD §7.10 practice" for n<10 cells but does not lock the 5-vs-4 boundary between descriptive-only and narrative-only tiers. The tier taxonomy in §9.2 is coherent and useful, but the citation attribution paints it as inherited machinery when it is in fact a new synthesis introduced here. Per [CONVENTIONS §2.2](../CONVENTIONS.md) methodology MD discipline, tier thresholds with downstream reporting consequences merit either a same-file rationale (four-input reasoning) or an inheritance pointer to a source that actually locks them. **Magnitude: substantive** because Stage D executors will read §9.2 as canonical tier definitions and cite them by attribution. **Absorb-vs-escalate**: mechanical-clarification absorb — either add a same-file "New tier terminology introduced by this audit; parent Stage -1 §7 uses n=5 as a single narrative-only marker on the 14-day drop" note, or move the tier introduction into a §5.5 sub-section owned by this audit rather than presenting it as parent-inherited. Safe for r1 lock with the note added.

**L1.2 — Substantive: descriptive-only-viable count is off by one.** §9.2 lists 5 cells in the 5<=n<10 tier (`lc_pre_ergo x +3d x strict x success`=6; `citalopram_modulated x +3d x strict x failure`=5; `citalopram_modulated x +10d x strict x success`=8; `pacing_habit_established x +3d x strict x failure`=8; `lc_pre_ergo x +10d x inclusive x failure`=5). The CSV output shows a sixth cell in that range: `pacing_habit_established x +5d x strict x failure`=6, which the audit itself lists elsewhere at §5.2 with the `(below)` marker. Under the audit's own tier definition (5<=n<10 = descriptive-only-viable), this cell belongs in the descriptive-only bucket, not the narrative-only bucket. If it moves to descriptive-only, narrative-only-viable drops from 15 to 14, so tier totals stay at 22+6+14+5=47 (should be 48) — I recount narrative-only cells 1<=n<=4 from the CSV as 15, so **22+5+6+15=48** balances with descriptive-only at **6** cells. Total-cell arithmetic (22 bootstrap + 5 n=0 + descriptive + narrative = 48) verifies. **Magnitude: substantive** — an off-by-one on a viability tier count that Stage D readers will use to plan cell disposition. **Absorb-vs-escalate**: mechanical-clarification absorb — add the missing sixth cell (`pacing_habit_established x +5d x strict x failure = 6`) to the descriptive-only list at §9.2 and correct "5 cells" to "6 cells"; narrative-only stays at 15. Safe for r1 lock post-correction.

**L1.3 — Minor: rounded means without decimals in §8 `total_steps` table.** §8 reports `total_steps` pre-window means as integers (6480 / 5500 / 5174 / 4816). The CSV values are 6479.83 / 5499.54 / 5174.43 / 4816.36; rounding to integers loses ~0.5 unit at the boundary between `lc_pre_ergo` and `pacing_pre_citalopram_learning` (6479.83 rounds to 6480, 5499.54 rounds to 5500). Not load-bearing for the monotone-decline finding (the ordering is stable at any reasonable rounding), but SCRIBE 2016 item 14 (report measures with appropriate precision) prefers one decimal on aggregate statistics. **Magnitude: minor**. **Absorb-vs-escalate**: mechanical absorb — add one decimal (6479.8 / 5499.5 / 5174.4 / 4816.4).

### Layer 2 fires (Daza 2018 + Personal Science norms)

**L2.1 — Substantive: stationarity assumption on personal baseline drift is not named.** The audit uses per-episode 30-day pre-windows for the detrended-arm eligibility rule (§6) and for the pre-window mean-level table (§8) without explicit acknowledgment that the pre-window mean-level baseline itself drifts across the 4-year LC-era corpus. §8's monotone `total_steps` decline (6480 → 4816 = 26% drop across phases) is precisely the corpus-level baseline shift that violates the stationarity assumption Daza 2018's counterfactual framing anchors on. The audit does frame the monotone decline as a "level-vs-change discipline requirement" per MD-alpha §3.5 (§9.4) which is correct downstream, but the Daza-anchored stationarity-of-baseline caveat is not named at §6 (where the 15-valid-pre-point rule is applied) or §8 (where the level shift is quantified). This matters because the "0 episodes drop from detrend-eligibility" finding at §6 is a favourable finding on **coverage**, but does not imply the linear-detrend-on-pre operation is well-formed on a per-episode 30d window when the underlying 4-year corpus is non-stationary. **Magnitude: substantive** because a Stage D executor reading §6's 0-drop finding as "detrend arm is fine" without §8's non-stationarity acknowledgment carries the risk. **Absorb-vs-escalate**: mechanical-clarification absorb — add a same-file cross-reference from §6's "favourable finding" language to §8's non-stationarity finding, e.g. a caveat sentence: "The 0-drop coverage finding is on the ≥15-valid-point rule only; it does not certify the per-episode linear-detrend-on-pre as well-formed on a corpus with the monotone pre-window level drift documented at §8. Per-episode detrend arm eligibility and cross-corpus baseline stationarity are separate conditions."

**L2.2 — Minor: held-out-structure framing not restated.** The audit inherits from parent Q24 MD which itself inherits the LC-era stratum. Per project memory `project_garmin_research_bias_boundary`, pre-2026 Garmin use was tactical-pacing, not analytical use — so the corpus itself has a Garmin-use-was-tactical caveat. Parent MD-alpha §3.6 lists this as the fifth confound in the phase-stratified conflation set. This audit is an inheritance-only Stage -1 audit and does not need to re-derive that caveat, but for a reviewer reading this audit standalone the tactical-Garmin-pacing caveat's absence risks a misread. §1's Inheritance-only clause + §3's "intensity-stratified arm does NOT structurally cluster" language could benefit from a single cross-reference to MD-alpha §3.6 fifth confound as a reminder that phase-stratified reads on this corpus interpret against a tactical-Garmin-use baseline that itself may have improved over the LC-era. **Magnitude: minor**. **Absorb-vs-escalate**: mechanical absorb — one-line pointer in §1 or §9.4 to MD-alpha §3.6 fifth confound.

### Layer 4 fires (project-specific audit hooks, [CONVENTIONS §§3-4](../CONVENTIONS.md))

**L4.1 — Substantive: `effective_exertion_min` at `pacing_pre_citalopram_learning` outlier flagged but not spike-vs-mean audited.** §8 correctly surfaces the `effective_exertion_min` pre-window mean at `pacing_pre_citalopram_learning` = 27.78 min/day as a level-vs-change disagreement signal against the other three phases' 5-19 min/day. §9.7 item 4 flags this for reviewer sanity. Per [CONVENTIONS §3.5](../CONVENTIONS.md#35-spike-detecting-metrics-over-daily-averages), spike-detecting metrics beat daily averages for sympathetic-arousal proxies — the flip side is that daily-mean outliers on a load metric can be a small number of high-load days dominating the mean. n=12 episode-ends x 30-day pre-windows = 360 pre-window days, but if the underlying `effective_exertion_min` pre-window distribution is right-skewed with a few 60-90 min days, the 27.78 mean can be 3-4 high-load days doing all the work. The audit reports mean AND median (28.42 min/day) which is close to the mean — that partially reassures the outlier is not a single-day artefact, and this is a favourable signal I should have flagged as a pass rather than a fire. The reviewer-sanity flag at §9.7 item 4 asks the right question ("axis-composition artefact or substantive envelope shift?"), but the audit could carry the mean-vs-median convergence (27.78 vs 28.42) into that item as evidence-in-hand that the outlier is not a small-n mean-inflation artefact. **Magnitude: minor** (the underlying analysis is correct; the framing at §9.7 item 4 undersells what the median tells us). **Absorb-vs-escalate**: mechanical-clarification absorb — add "mean 27.78 close to median 28.42 rules out small-n mean-inflation as the outlier driver" to §9.7 item 4.

**L4.2 — Substantive: `pacing_pre_citalopram_learning` phase-day-count vs episode-count discipline blurred at §9.7 item 2.** §9.7 item 2 says "at n=12 combined episode-ends, this phase sits at the boundary of every viability tier". Per [CONVENTIONS §3.6](../CONVENTIONS.md#36-name-every-count-n-unit-per-scheme-in-file) named-counts discipline, this should specify unit + scheme + source in the same sentence. The audit does provide named-counts panels per section (e.g. §2 "scheme = combined heavy-episode-ends counted at the last day of a gap=0 contiguous run; unit = episode-ends; source = `output/phase_episode_counts.csv`"), so the discipline is followed **within-section**, but §9.7 item 2 references the count without repeating the scheme. Not a fire because the surrounding context establishes the scheme in §2; still worth an inline "(combined heavy-episode-ends, gap=0)" for reader-scan robustness at §9.7. **Magnitude: minor**. **Absorb-vs-escalate**: mechanical absorb — add scheme parenthetical to §9.7 item 2.

**L4.3 — Pass: `_lagged_lcera` variant used correctly.** The script (`scripts/audit.py` line 91) correctly reads `exertion_class_lagged_lcera` for the heavy-day classification, matching [CONVENTIONS §3.2](../CONVENTIONS.md#32-lagged-baseline-for-sustained-push-hypotheses) — the LC-era-restricted lagged variant is the right choice for PEM-pacing analysis on `lc_phase == 'lc'` frame. No fire.

**L4.4 — Pass: zero-vs-NaN discipline held.** The script never applies `.fillna(0)` to any outcome column. §4 coverage reports report `n_nan_outcome_at_d_plus_k` explicitly alongside `n_valid`. §5.4 identifies structurally-unavailable cells (n=0) separately from valid-but-small cells and explicitly instructs Stage D to emit NaN not zero. §7.5 named-counts panels are unit-explicit. This is a strong pass on [CONVENTIONS §5](../CONVENTIONS.md#5-project-wide-anchors-read-once-then-trust) zero-vs-NaN + [CONVENTIONS §3.6](../CONVENTIONS.md#36-named-counts) — worth noting in §3 below.

**L4.5 — Pass: caveat-class discipline held per [CONVENTIONS §4.2](../CONVENTIONS.md#42-caveats-yes-a-priori-claims-no).** §8 and §9.4 frame both the `total_steps` monotone decline and the `effective_exertion_min` outlier as "read-discipline requirements" not "a-priori claims about pacing-learning". The wording explicitly says "not a caveat rebuttal — a discipline requirement" and "Neither finding is a caveat rebuttal — both are Stage D read-discipline requirements". This is the KEEP-class caveat framing done well; no fire.

## 3. What does not fire (selective, non-trivial passes only)

**Reproducibility byte-match verification held under fresh probe.** §2 and §3 reproduce MD-alpha §3.1 (19 / 12 / 125 / 158 / 314) and §6.3 (heavy 8/4/70/83=165; very_heavy 11/8/55/75=149) byte-identically. I verified the phase-count total against the CSV output; no divergence. This is a favourable audit-execution finding — the operand definition in MD-alpha is well-formed on the CSV state as of 2026-07-16, and the audit provides the peer-review probe that would have caught any drafted-time-vs-audit-time drift.

**Inheritance-only status held cleanly.** The audit's §1 Inheritance-only clause is not merely stated — the audit follows it. Corpus counts (§1), heavy-day definition, episode structure, overlap density, and intensity stratification design decisions all point at parent Stage -1 audit sections or parent MD sections rather than re-deriving. The one place it could be tighter (§9.2 tier taxonomy) is called out at L1.1 above; otherwise inheritance discipline is exemplary.

**Sample-floor discipline held under my recount.** 26 / 48 below-floor cell count reproduces on the CSV. 5 structurally-unavailable cells (n=0) are correctly enumerated at §5.4 and correctly distinguished from valid-but-small cells at §9.2. This matters because a reader could have conflated n=0 with n=1 as "narrative-only" — the audit's separation is disciplined.

**Level-vs-change confound on `total_steps`.** §8's monotone pre-window `total_steps` decline (6480→5500→5174→4816) is correctly framed as a level-vs-change confound per MD-alpha §3.5, NOT as pacing-learning evidence. §9.4 walks the reader through the discipline requirement (report pre-window mean panel alongside AUC panel; do not attribute monotone AUC-magnitude trend to pacing-learning without acknowledging the parallel monotone level decline). This is the correct handling of a signal that could otherwise contaminate the sub-part 3 rescue-subset call.

**Detrended-arm parity finding held.** §6's "0 episodes drop from ≥15-valid-pre-point rule across all 16 phase x outcome cells" is a real finding (CSV verified). The audit's Read of that finding is disciplined: "Whatever sample-size compression happens at §5 applies identically to raw and detrended arms; no additional per-outcome per-phase detrend-drop needs to be modelled." This is not silently masking a real filter concern — the audit reports the caveat at the boundary edge (§6 caveat paragraph + §9.7 item 5): "No episode-end in the four phases has a pre-window shorter than 15 days in the corpus", verified against phase-membership pattern. Boundary condition (pre-window truncation for episode-ends near LC-era start) is explicit.

**`pacing_pre_citalopram_learning` outlier surfaced not absorbed silently.** §8 reads `effective_exertion_min` = 27.78 min/day at that phase as an outlier vs 5-19 elsewhere; §9.4 and §9.7 item 4 both surface it as a reviewer concern rather than a routine number. The mean-vs-median (27.78 vs 28.42) reads consistent within-phase, ruling out a single-day mean-inflation artefact — the outlier is a real phase-level baseline finding. The audit is calling this out for Stage D + reviewer attention rather than absorbing it into the pre-window mean-level table without comment.

**Rescue-metric structurally undefined at +10d strict-clean × success surfaced correctly.** §9.3 correctly identifies that MD-alpha §3.5 Spearman ρ is undefined at +10d strict-clean × success because `pacing_pre_citalopram_learning` at n=0 makes one of the four phase-cells NaN. §9.7 item 7 forces Stage D to run the rescue metric only where all four phases have n>0 — i.e. +3d strict-clean × success and +5d strict-clean × success and all inclusive-policy windows. This is a load-bearing finding that saves Stage D from computing an undefined statistic.

**Discipline concerns list (§9.7) is well-formed and actionable at Stage D.** Seven items, each with a specific mechanism (n=0 NaN emission; small-phase leverage; level-vs-change confound; non-monotone baseline; boundary-truncation caveat; overlap-policy divergence; rescue-metric NaN). Each item names an actionable Stage D discipline (emit NaN not zero; treat with caution; report pre-window mean panel; flag for reviewer sanity; re-verify if corpus boundary changes; report both policies; do not attempt rescue metric at +10d strict). Item 4 could be sharpened per L4.1 above (mean-vs-median convergence as favourable evidence-in-hand); otherwise concerns are well-scoped.
