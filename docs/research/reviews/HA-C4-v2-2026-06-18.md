### Side observations

- **Side** — v2 §5.3 closing paragraph asserts *"v1's §5.3 implicitly assumed every channel × era cell yields SUPPORTED or REFUTED; an INCONCLUSIVE cell broke the binary triad logic."* This slightly overstates v1's gap: v1 §5.3 actually included a 0.5-single-era-confirmation mechanism ("2.0 → SUPPORTED, 1.5 → PARTIAL+, 1.0 → PARTIAL, 0.5 → PARTIAL−, 0 → REJECTED"). The actual ambiguity v2 resolves is the interaction of v1 §5.4 ("INCONCLUSIVE channels do not count toward the triad verdict") with v1 §5.3's 2-of-3 rule when a channel had one SUPPORTED era and one INCONCLUSIVE era. v2's CONFIRMED-PARTIAL category resolves a real ambiguity, but the v2 diagnostic framing of *what* v1 lacked is imprecise. Not a fire — the substantive resolution is right; one-line re-phrasing would tighten the honest-framing audit trail.
- **Side** — §5.5 notes *"Holm cannot be computed on a cell that is INCONCLUSIVE (no p-value); that cell is omitted from the Holm step-down ordering."* Holm with fewer comparisons is structurally less stringent. The §5.0 triad-sum verdict is the binding bar; Holm is secondary report only — so this is not a multiplicity-discipline fire — but the result.md author should name the omission explicitly when the Holm row is written so the reader does not read a 2-channel-Holm as 3-channel-Holm.

- **Side** — §4.11.3's *"non-heavy-T arm size adjusted to match (chain-relaxed pairing)"* phrasing is under-specified for the test.py writer: it does not state whether "match" means re-pairing per heavy-T day (1:N matching) or simply taking the chain-relaxed pool's natural non-heavy-T count.
