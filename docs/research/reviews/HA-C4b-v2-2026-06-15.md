## 1. What the data shows

The target is a **v2 pre-registration** (not a result). It states, before any v2 test run, the locked operationalisation for a Wiggers-C4-style precursor test on the new Session E `stress_low_motion_min_count_S60_Mlow` primitive, with the v1 headline cell relocked after v1's same-day dry-run halt:

- **v2 headline claim**: among `crash_v2` episodes whose lead-up window includes a heavy / very_heavy `exertion_class_lagged_lcera` day, at least one day's `stress_low_motion_min_count_S60_Mlow` deviates from the lagged same-phase baseline by `z >= 1.5` (one-sided elevated), and the crash-episode frequency of this trigger exceeds the matched stationary-bootstrap-null frequency on the **unmedicated phase, train + validate eras pooled** (n = 10).
- **v1 → v2 reframe**: v1 locked the headline at consolidation × both-eras-independently (n = 0 train + n = 5 validate, both unsatisfiable / inconclusive); v2 relocks at unmedicated × train+validate pooled (n = 10, just clearing the §5.3 bar). The both-eras-independent rule is abandoned for the v2 headline cell; cross-era directional-consistency descriptive companions on the train-only (n = 8) and validate-only (n = 2) subsets partially compensate.
- **v2-specific addition**: §4.11.5 episode-level leave-one-out (LOO) fragility check. At n = 10 a single episode is worth 10 percentage points on the §5.1 (a) gate, so verdict stability is structural; LOO reports a range and flags load-bearing episodes. Descriptive only, no SUPPORTED-bar weight.

The empirical claim is the same trigger-frequency-vs-null test as v1 (and the HA11 / HA06b / HA10 family); the interpretive framing — "stuck sympathetic / walls of orange while body is at rest" — held separate from the trigger mechanics. The v1 audit (`reviews/HA-C4b-2026-06-15.md`) verdict was REVISION RECOMMENDED; v1 r2 + r3 closed those fires; v2 inherits the closed-state baseline and adds the four v2 deltas above.

## 4. What would strengthen this finding

1. **Tighten the §2 unmedicated argument**. Add 2-3 sentences immediately after the §2 wrapper sentence that explicitly trace *why each inherited v1 motivation still favours unmedicated as the headline phase* (e.g. "Wiggers' C4 protocol assumes uninhibited sympathetic regulation; the §5.6-confirmed citalopram stress-suppression cascade therefore makes the unmedicated phase the cleanest test ground. The four sibling SUPPORTED autonomic-deviation precursors that motivated this test (H02b train, H02d train, HA06b train, HA11 train) all fired in pre-cliff windows that overlap the unmedicated phase substantially — the precursor evidence base is unmedicated-leaning, not consolidation-leaning"). This converts the §2 wrapper sentence into a self-contained argument and removes the soft Layer 1 framing concern. The §1 reframe paragraph + §4.4 v2-headline-verdict paragraph + §8 caveat already carry the explicit argument; the §2 polish is the closure for self-readability.
2. **Anchor the §7 σ-range lower bound** in v2 §7. Replace "the lagged baseline (60-day window) typically narrows this somewhat" with "the lagged baseline (60-day window) typically narrows this somewhat — v1 dry-run observed σ_median = 33.4 on §4.3-eligible unmedicated days; the [25, 55] range brackets this margin below and the raw 53.6 just above". One-line fix; closes the side observation.
3. **Add a §4.11.5 boundary-fragility note** (5-line addition). State explicitly that the LOO load-bearing-episode list is most informative at exactly the §5.1 (a) 60% boundary (k = 6 firing episodes out of 10) — at k = 7+ or k = 5-, no LOO drop flips the verdict, so the load-bearing list will be empty and the LOO range will be tight. This is a structural observation about the n = 10 boundary, not a fragility of the v2 design; surfacing it pre-emptively helps the result.md reader interpret the LOO output correctly.
4. **§4.3 + §4.11 remaining external-viz citations** could be optionally consolidated into a single in-repo summary file (e.g. `analyses/garmin_exploration/stress_low_motion_viz/key-findings.md` with the ρ = 0.79 and ρ = 0.556 numbers + the Family A 2024-11-26 case description) so the next fresh-session reviewer can verify the cited values without leaving the repo. Not load-bearing for v2 — the §7 anchor (the most analytically critical citation) is already in-repo via the script — but completes the v1 audit's side-(c) closure for future audits.
